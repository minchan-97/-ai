# Logic AI Contract Reviewer

계약서 조항별 두 관점(중립 설명 vs 위험 검토) 일관성 점검 도구.

## ⚠️ 이 도구가 무엇이고 무엇이 아닌가

**무엇인가**
- LLM consistency 체크 도구의 계약서 도메인 변주
- 두 관점이 갈리는 조항을 사용자가 직접 더 면밀히 검토하도록 안내
- 본인 계약서를 스스로 검토할 때의 보조 도구

**무엇이 아닌가**
- 변호사 자문 대체 **아닙니다**
- 법적 효력·유효성 판정 도구 **아닙니다**
- 모든 위험을 자동 탐지하는 도구 **아닙니다**
- 계약 체결 결정의 단독 근거 **될 수 없습니다**

계약 체결 전에는 반드시 변호사나 법무 전문가의 자문을 받으십시오.

## 동작 원리

1. 계약서 텍스트를 입력받아 정규식 기반으로 조항 단위 분할(`제N조`, `Article N` 등)
2. 각 조항에 대해 LLM을 두 번 호출:
   - **중립 설명** system prompt: 조항이 규정하는 권리·의무를 객관적으로 설명
   - **위험 검토** system prompt: 모호함·일방성·면책·해지 등 잠재 위험 식별
3. 두 응답을 OpenAI 임베딩으로 벡터화 → cosine distance 0~100으로 환산
4. 임계값(기본 20%) 초과 조항을 "INCONSISTENT(검토 우선)"로 표시
5. 두 응답을 나란히 보여줘서 사용자가 직접 비교

## 설치 / 실행

```bash
pip install -r requirements.txt
streamlit run contract_app.py
```

`.streamlit/secrets.toml` 또는 환경변수:

```toml
OPENAI_API_KEY = "sk-..."
OPENAI_MODEL = "gpt-4o-mini"
OPENAI_EMBEDDING_MODEL = "text-embedding-3-small"
```

## 정직한 한계

- **조항 분할은 휴리스틱**. 비표준 양식 계약서는 잘못 분할될 수 있음. 분할 실패 시 전체를 1개로 처리.
- **두 응답이 비슷해도 둘 다 같이 빠뜨릴 수 있음** (공통 환각).
- **두 응답이 달라도 반드시 문제 있는 조항은 아님**. 검토 우선순위 신호일 뿐.
- **임계값은 데이터로 검증되지 않은 휴리스틱**. 도메인에 맞게 조정 필요.
- **비용**: 조항당 chat 2회 + embedding 2회. 50개 조항이면 chat 100회 + embedding 100회. gpt-4o-mini + text-embedding-3-small 기준 약 25원 내외.

## 비용 절감 팁

- `cloud_features` (v4 일반 앱의 N회 호출 옵션)는 **본 앱에서 의도적으로 제외**했습니다. 조항당 N=5 호출은 비용 5배가 됩니다.
- 큰 계약서는 먼저 임계값을 높여 INCONSISTENT만 빠르게 식별한 뒤, 그 조항만 다시 낮은 임계값으로 살펴보는 게 효율적입니다.

## 시장 위치 (정직한 자기 평가)

이 도구는 한국 법률 LLM 시장의 다음 자리와 **겹치지 않습니다**:

- **로웨이브, SQI SOFT 등 상용 RAG 변호사 도구**: 변호사용, 출처 인용 검증
- **korean-law-mcp**: 법령 조문 실존 여부 교차검증

이 도구가 채우는 자리:
- **사용자가 본인이 계약서를 읽을 때**의 일관성 보조
- 작은 회사 법무 담당, 스타트업 창업자, 부동산 거래 전 일반인 등
- 변호사 자문 받기 전 1차 자가 검토

## 파일 구조

```
contract_app.py     — Streamlit 메인 (계약서 전용 진입점)
contract.py         — 조항 분할 + 두 관점 system prompt
safety.py           — 누적 안전 차단 (v4와 공유, 선택)
cloud_features.py   — v4 공유 (본 앱에서 미사용)
app.py              — v4 일반용 (Q&A 형식, 본 앱과 독립)
```

`contract_app.py`와 `app.py`는 같은 디렉토리에 공존하며, 서로 독립적으로 실행 가능합니다.
